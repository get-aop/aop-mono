## Why

This is a **planning change** - not an implementation change. It defines the overall MVP vision and architecture for the AOP (Agents Operating Platform) across multiple milestones.

## What Changes

This change produces a comprehensive design document (`design.md`) that serves as the roadmap for building the AOP platform. It does not directly implement code.

## Capabilities

### New Capabilities

- `mvp-design`: Comprehensive design document defining architecture, decisions, and 5-milestone build plan for the AOP platform

### Modified Capabilities

None - this is a planning artifact.

## Impact

- **New artifact**: `design.md` - The MVP roadmap referenced by all implementation milestones
- **No code changes**: This is planning work, implementation happens in separate changes (aop-foundation, etc.)

## Note

This change follows a **design-first** approach where the design artifact is the primary deliverable. Implementation is broken into separate changes (Milestone 1: aop-foundation, etc.) that reference this design.
